Como organizar o CSS?

O site já tem um CSS bem elaborado com variáveis, temas e responsividade. Como queremos estruturá-lo no React?

A

CSS Global + Módulos por componente

Variáveis globais e reset em global.css. Cada componente tem seu próprio Component.module.css com estilos isolados.

Prós

  • Migração direta do CSS atual
  • Zero colisão de nomes
  • Padrão React moderno

Contras

  • Mais arquivos para manter
B

CSS Global único (como está hoje)

Todos os estilos num único styles.css como no HTML atual. Migração mais direta, sem refatoração de CSS.

Prós

  • Migração mais rápida
  • CSS já funciona bem

Contras

  • Estilos globais podem colidir
  • Difícil de escalar
C

Tailwind CSS

Trocar todo o CSS por utilitários do Tailwind. Design system consistente, classes direto no JSX.

Prós

  • Desenvolvimento rápido após aprender
  • Fácil de manter consistência

Contras

  • Reescrita completa do CSS atual
  • JSX fica verboso com muitas classes
  • Curva de aprendizado

Recomendação: A (CSS Global + Módulos) — aproveita o CSS existente nas variáveis globais e isola cada componente no seu próprio arquivo.

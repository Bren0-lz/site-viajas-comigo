Qual estrutura React usar?

A escolha afeta roteamento, SEO, deploy e complexidade do projeto.

A

Vite + React (SPA)

App de página única com React Router. Simples, rápido de configurar, hot reload instantâneo. Deploy estático (Netlify, Vercel, GitHub Pages).

Prós

  • Mais simples de configurar
  • Deploy estático fácil
  • Perfeito para o projeto atual
  • Admin integrado na mesma app

Contras

  • SEO limitado (sem SSR)
  • Primeira carga pode ser mais lenta
B

Next.js (SSR/SSG)

Framework full-stack com renderização no servidor. Melhor SEO, rotas automáticas por arquivo. Ideal se quiser adicionar backend futuramente.

Prós

  • SEO excelente (páginas de viagens indexáveis)
  • Estrutura escalável
  • App Router moderno

Contras

  • Mais complexo para começar
  • Deploy requer Node server (ou Vercel)
  • Curva de aprendizado maior

Recomendação: A (Vite + React) — o projeto é 100% client-side com localStorage, sem necessidade de SSR no momento. É possível migrar para Next.js depois se precisar.
